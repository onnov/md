Пример структуры проекта:

    backend/
    ├── api/
    │   └── spec.yaml          # OpenAPI спецификация
    ├── internal/
    │   ├── handler/           # Обработчики (генерируются ogen)
    │   ├── storage/
    │   │   ├── file/          # Реализация для файлового хранилища
    │   │   └── postgres/      # Реализация для БД (опционально)
    │   └── service/           # Бизнес-логика
    ├── data/                  # Файлы с фейковыми данными
    │   ├── products.json
    │   ├── users.json
    │   └── orders.json
    └── go.mod

Ключевые моменты для учебного проекта:

  • Простота данных - готовые фейковые наборы для быстрого старта
  • Четкая документация - OpenAPI спецификация как единый источник истины
  • Модульность - возможность постепенного добавления функциональности
